# 仕様整合チェック（question_update_flow）

次の変更について、`docs/question_update_flow_specification.md` と突き合わせて整合性を確認してください。

## 対象

- 今回触ったファイル・差分の要約

## 出力

1. **仕様書の該当セクション**（見出し名でよい）
2. **一致している点**（箇条書き）
3. **未記載・あいまいな点**や、仕様を更新すべき差分があれば明示
4. 問題データや管理コマンドを変えた場合は、**登録コマンド名**と **data/questions のファイル**の対応が正しいか

仕様書にない独自判断をした場合は、その理由を一文で書いてください。
