# ゲーミフィケーション ロードマップ・進捗管理

## 概要

英検学習アプリ（主な利用者: 小学生・中学生）向けに、**学習継続**と**次に何をすればよいかの明確化**を目的としたゲーミフィケーションを段階的に導入する。

### 設計方針

| 原則 | 内容 |
|------|------|
| 学習が主、ゲームは添え物 | ポイント競争より「本番に近づく実感」を優先 |
| 努力を褒める | 正答率だけでなく「今日やった」「続けている」も価値にする |
| 失敗を怖くしない | 全体ランキング・厳しいストリーク切れは避ける |
| 既存進捗データを活かす | `UserProgress` / `DailyProgress` / 解放ロジックを再利用する |

### 対象外（当面）

- 他ユーザーとの公開ランキング
- ポイントショップ（見た目だけの報酬）
- 進捗ページ（`progress.html`）の Chart.js 詳細グラフの置き換え（保護者向けとして残す）

## 既存実装（Phase 1 完了分）

### 1. セッション結果の達成メッセージ

**画面**: 回答結果（`exams/templates/exams/answer_results.html`）

**ロジック**: `exams/gamification.py` → `build_session_achievements()`

**表示例（最大2件）**

- 全問正解 → 「パーフェクト！ぜんぶ正解だよ」
- 80%以上 → 「よくがんばった！あと一歩」
- 50%以上 → 「いい調子！間違えたところを見直そう」
- 今日初の学習 → 「今日のスタート、ナイス！」
- ランダム10問 / 模擬試験が新規解放 → 解放お知らせ
- 模擬試験に近いカテゴリ → 「模擬試験まであと○%（カテゴリ名）」（残り40%以内）
- ライティング → 「提出おつかれさま！…」（自動採点しない旨）

**技術メモ**

- 提出直前に `_snapshot_unlock_before_submit()` で解放状態をセッション保存
- 結果表示時に `pop_pre_submit_unlock_snapshot()` で比較し「新規解放」を検出
- スナップショット挿入箇所: `submit_answers` / `submit_reading_comprehension` / `question_list` の POST 各分岐

### 2. プログレスバー強化（問題一覧）

**画面**: 問題一覧（`exams/templates/exams/exam_list.html`）

**コンポーネント**

- 各問題形式ボタン下: `exams/templates/exams/_foundation_progress.html`
- 総合問題の上: 「冒険の進み具合」カード（`build_adventure_summary()`）

**表示内容**

- 基本・リスニング各カテゴリ: 取り組み率バー + 「模擬試験まであと○%」（80%未満）
- 3級ライティング: 取り組み率のみ（模擬試験の解放条件には含まれない旨を表示）
- 冒険カード: ランダム10問（3カテゴリ×20%）、模擬試験（全カテゴリ×80%）の進捗

**解放条件（変更なし・表示のみゲーム化）**

| 機能 | 条件 |
|------|------|
| ランダム10問 | 基本問題のうち **3カテゴリ以上** が取り組み率 **20%以上** |
| 模擬試験 | `FOUNDATION_QUESTION_TYPES` の **全カテゴリ** が **80%以上** |

定数: `exams/views.py` の `RANDOM_UNLOCK_*` / `MOCK_EXAM_UNLOCK_MIN_RATE`  
集計: `_build_exam_unlock_status()` / `_progress_rate_for_type()`

## 関連ファイル

```
exams/
├── gamification.py              # メッセージ・サマリ・スナップショット
├── views.py                     # unlock 集計、exam section、answer_results
├── models.py                    # UserProgress, DailyProgress
└── templates/exams/
    ├── exam_list.html           # 問題一覧 + 冒険カード
    ├── answer_results.html      # 達成バナー
    └── _foundation_progress.html  # カテゴリ別バー
```

**進捗の一次データ**

- `UserProgress`: ユーザー × 級 × 問題タイプ（累計・正答数・最終回答日）
- `DailyProgress`: 日次の取り組み数
- 取り組み率の分子: カテゴリごとに「回答済み distinct 件数」（読解は本文単位など）

## 参考: 画面フロー（Phase 1 完成時点）

```
ログイン → 問題一覧
  ├ 各形式ボタン + 取り組み率バー
  ├ 冒険の進み具合（ランダム / 模擬）
  └ 総合問題（解放済みなら有効）

問題解答 → 回答結果
  ├ 正解数
  ├ 達成メッセージ（0〜2件）  ← Phase 1a
  └ 解説 → 問題一覧へ

学習進捗（別ナビ）
  └ グラフ・詳細テーブル（保護者向け・現状維持）
```
